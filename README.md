# Bonus Creative

Premium website project for an Oklahoma-based solo filmmaker and video producer.

## What this repo is for
This repository is the foundation for a new website for Bonus Creative. The site should feel cinematic, editorial, minimal, and fast, while making it easy for clients to understand the value of working directly with one filmmaker from concept to final cut.

## Brand position
Bonus Creative is not a generic agency. It is a filmmaker-led studio built around:

- one creator
- one vision
- direct collaboration
- cinematic quality
- lean execution
- story-first thinking
- premium results without agency bloat

## Primary audiences
- Nonprofits
- Mission-driven organizations
- Churches
- Brands
- Founders
- Conferences and events
- Corporate teams that want a filmmaker, not a bloated agency

## Core site goals
1. Make the work feel cinematic and credible immediately
2. Explain why direct collaboration with one filmmaker is a strategic advantage
3. Showcase reel, work, and case studies clearly
4. Convert visitors into project inquiries
5. Load extremely fast on mobile and desktop

## Suggested sitemap
- Home
- Work
- Services
- About
- Case Studies
- Contact
- Optional: Reel, FAQ, Journal

## Design direction
- Dark, elegant, minimal
- Strong typography
- Large visual rhythm
- Editorial layout language
- Restrained motion
- Premium but approachable
- No gimmicks, no clutter, no agency-template feel

## Suggested tech direction
- Astro or Next.js
- Static-first rendering
- Vercel, Netlify, or Cloudflare Pages
- Responsive AVIF/WebP images
- Click-to-load video instead of heavy embeds
- Minimal JavaScript
- Excellent Core Web Vitals

## Repo structure
- `docs/brand-brief.md` — cleaned strategic brief
- `docs/original-brief.md` — source brief from conversation
- `docs/sitemap.md` — content architecture by page
- `docs/copy-framework.md` — copy guidance by section
- `docs/case-study-template.md` — reusable project page structure
- `docs/performance-seo.md` — speed and SEO standards
- `docs/mobile-notes.md` — mobile-first interaction notes
- `public/` — future media assets
- `src/` — future site code

## Best next step
Choose Astro or Next.js, then build the homepage first using the guidance in `/docs`.
